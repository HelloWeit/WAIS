# Brainstorm Agent Skill

## 目标

为技术管理者/架构师设计的交互式探索工具，**所有输出保存到 Inbox**，后续通过工作流处理。

## 用户角色

- 架构师
- 技术经理
- 技术专家
- 技术团队负责人

## 定位

```
工作流：外部信息 → Inbox → Brief → Task/Project → Status → Execution → Research → Knowledge → Review
                       ↑
                 /brainstorm 输出到这里
```

**核心原则：** 所有探索结果保存到 Inbox，后续通过 `/task` 或 `/project` 处理。

## 使用场景

- 探索技术架构方向
- 思考团队组织问题
- 分析技术选型决策
- 挖掘创新机会
- 梳理复杂问题

---

## Phase 1: 交互探索

### 探索角度（针对技术管理者）

1. **技术维度**
   - 架构设计
   - 技术选型
   - 性能/扩展性
   - 技术债务

2. **团队维度**
   - 资源分配
   - 能力建设
   - 协作效率
   - 跨团队协调

3. **战略维度**
   - 业务对齐
   - 竞争优势
   - 长期规划
   - 风险评估

4. **创新维度**
   - 新技术机会
   - 效率提升
   - 自动化可能

### 探索技术

- **5 Whys**：深入挖掘根本原因
- **What if?**：探索替代方案
- **权衡分析**：利弊对比
- **场景推演**：不同场景下的表现
- **约束思考**：资源/时间/人力约束

### 对话流程

1. **理解上下文**
   - "这个想法的背景是什么？"
   - "想解决什么问题？"
   - "有什么约束条件？"

2. **深入探索**
   - 从多个维度提问
   - 挑战假设
   - 探索替代方案

3. **捕获洞察**
   - 关键发现
   - 决策点
   - 风险点
   - 行动方向

---

## Phase 2: 综合并输出到 Inbox

探索完成后，生成结构化笔记，**保存到 Inbox**。

### 输出文件

- **目录：** `00_Inbox/`
- **文件名：** `IDEA_YYYYMMDD_短标题.md`

### 内容结构

```markdown
---
type: idea
status: pending
created: YYYY-MM-DD
source: brainstorm
category: architecture|team|strategy|tech|opportunity
---

# [想法标题]

## 核心想法
[一段话总结]

## 探索洞察

### 技术维度
- [洞察1]
- [洞察2]

### 团队维度
- [洞察]

### 战略维度
- [洞察]

## 可能方向
1. **方向A**: [描述] - [优缺点]
2. **方向B**: [描述] - [优缺点]

## 关键决策点
- [ ] 决策1: [描述]
- [ ] 决策2: [描述]

## 风险/约束
- [风险1]
- [约束1]

## 下一步建议
- `/task` - 转为可执行任务
- `/project 00_Inbox/此文件.md` - 启动为项目
- `/research` - 深入研究某方面
```

```
## 想法已保存

- `/task` - 转为可执行任务
- `/project 00_Inbox/IDEA_xxx.md` - 启动为项目
- `/research` - 深入研究
- 或等待下次 Morning Brief 处理

**Inbox 状态:** [N] 条待处理
```

---

## 与工作流的整合

```
/brainstorm 探索想法
     ↓
交互式探索（技术/团队/战略维度）
     ↓
保存到 00_Inbox/IDEA_xxx.md
     ↓
（后续路径 - 保持工作流一致性）
     ├─→ /task → 20_Tasks/（明确任务）
     ├─→ /project → 30_Projects/（完整项目）
     ├─→ /research → 40_Research/（深度研究）
     └─→ Morning Brief → 识别处理
```

---

## 约束

- **必须**输出到 `00_Inbox/`
- **禁止**直接创建任务或项目
- **禁止**跳过 Inbox
- 保持工作流一致性：想法 → Inbox → 分类处理

---

/brainstorm AI 技术在业务中的应用
/brainstorm 技术债务治理策略
